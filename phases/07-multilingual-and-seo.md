# Phase 07 — Multilingual And SEO

## Цель

Подготовить сайт к языкам: русский, английский, испанский, португальский, французский, немецкий. URL должны быть под папками.

## Рекомендованный подход

Использовать готовый WordPress-плагин мультиязычности:

- Polylang Pro — легче, хорошо подходит для folder-based URL.
- WPML — мощнее для крупных проектов и переводческих workflows.

Для MVP я бы начинал с Polylang Pro, если нет жестких требований к WPML.

## URL Structure

- `/ru/`
- `/en/`
- `/es/`
- `/pt/`
- `/fr/`
- `/de/`

## Что переводим

- UI-строки темы и плагина через gettext.
- Новости как обычный переводимый content.
- Описания букмекеров вручную/через перевод.
- SEO title/description для всех сущностей.
- Названия стран/позиций/статусов через словари.

## Что не обязательно переводить вручную

- Имена игроков.
- Названия команд.
- Числовая статистика.
- Счет, даты, турнирные таблицы.

## SEO

- ЧПУ для всех сущностей.
- `hreflang`.
- Schema.org:
  - `NewsArticle`;
  - `SportsEvent`;
  - `SportsTeam`;
  - `Person`;
  - `Review` для букмекеров, если применимо.
- XML sitemap по языкам.
- Canonical URL.

## Результат фазы

- Выбран плагин мультиязычности.
- Настроены языковые папки.
- Подготовлены gettext-строки.
- Базовая SEO-структура.
