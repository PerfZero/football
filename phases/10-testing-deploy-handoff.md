# Phase 10 — Testing, Deploy And Handoff

## Цель

Подготовить проект к передаче, продакшену и дальнейшей поддержке.

## Проверки

- PHP syntax.
- WordPress debug log.
- Проверка пустых API-ответов.
- Проверка лимитов API.
- Проверка multilingual URL.
- Проверка canonical/hreflang.
- Адаптивность основных страниц.
- Скорость загрузки.
- Кеширование.
- Безопасность хранения API key.

## Performance

- Не ходить в API при каждом просмотре страницы.
- Использовать object cache/transients/custom tables.
- Оптимизировать изображения.
- Lazy-load новостей/списков при необходимости.
- Отдельно продумать live-матчи, чтобы не перегружать API.

## Deploy

- Подготовить production `.env`.
- Развести dev/stage/prod.
- Описать запуск cron.
- Описать бэкапы базы.
- Описать обновление темы и плагина.

## Документация

- Как запускать Docker.
- Как менять тему.
- Как вводить API key.
- Как запускать ручную синхронизацию.
- Как добавлять букмекера.
- Как добавлять/переводить новость.
- Как обновлять языковые строки.

## Результат фазы

- Готовый к передаче проект.
- Документация для разработчика.
- Документация для контент-менеджера.
