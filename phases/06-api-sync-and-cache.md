# Phase 06 — API Sync And Cache

## Цель

Подготовить надежную синхронизацию с API-Football без лишнего расхода лимитов и без торможения страниц.

## Принцип

Фронтенд не должен ждать API. Страницы читают локальные данные WordPress/кеша. API вызывается по cron, вручную из админки или точечно при обновлении.

## Кеширование

- Reference data: кешировать надолго или обновлять раз в неделю.
- Leagues/teams/squads: обновлять ежедневно.
- Fixtures на ближайшие дни: обновлять чаще.
- Live fixtures/events/statistics: обновлять короткими интервалами только для live-матчей.
- Odds pre-match: обновлять по расписанию перед матчами.
- Odds live: только если нужна live betting-логика.

## Sync Jobs

- `sync_reference_data`
- `sync_leagues`
- `sync_teams`
- `sync_players`
- `sync_fixtures`
- `sync_fixture_details`
- `sync_standings`
- `sync_odds_bookmakers`
- `sync_odds`

## Mock Mode

Пока API-ключа нет:

- создаем демо-JSON рядом с плагином;
- используем те же DTO/мапперы, что и для API;
- тема сразу работает с реалистичной структурой;
- после получения ключа меняется только источник данных.

## Ошибки и лимиты

- Логировать код ответа API.
- Сохранять последнюю успешную версию данных.
- Не ломать страницу при пустом ответе.
- Показывать в админке статус последней синхронизации.

## Результат фазы

- Рабочий мок-режим.
- Базовые sync jobs.
- Кеш-слой.
- Таблица/экран логов API.
