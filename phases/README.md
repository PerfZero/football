# Football Project Phases

Этот каталог фиксирует план разработки WordPress-сайта Football: тема, API-плагин, шаблоны страниц, мультиязычность и подготовка к реальным данным API-Football.

## Главная идея

Делаем не просто тему, а связку:

- `football` theme: внешний вид, шаблоны, компоненты, адаптив.
- `football-data` plugin: настройки API, загрузка/кеширование данных, CPT, таксономии, синхронизация, shortcodes/blocks, cron jobs.
- multilingual layer: URL под папками `/ru/`, `/en/`, `/es/`, `/pt/`, `/fr/`, `/de/`.

Пока API-ключа нет, поэтому в первых фазах используем мок-данные, структуру API и демо-JSON. Когда ключ появится, плагин переключится с моков на реальные запросы.

## Минимальный набор шаблонов

- Главная.
- Новости: список и отдельная новость.
- Турнир/чемпионат: обзор, таблица, матчи, команды, игроки, новости.
- Матч: счет, события, составы, статистика, коэффициенты.
- Игрок: профиль, статистика, матчи, карьера, достижения, новости.
- Команда/участник турнира: профиль, состав, матчи, таблица, новости.
- Букмекеры: рейтинг/список.
- Отдельная страница букмекера.
- Статистика: лидеры турниров, команды, игроки.

## Порядок фаз

1. `01-discovery-and-scope.md` — уточнение объема и карты сущностей.
2. `02-data-plugin-architecture.md` — архитектура API-плагина и моделей данных.
3. `03-wordpress-content-model.md` — CPT, таксономии, URL и админка.
4. `04-theme-ui-system.md` — дизайн-система темы по скриншотам.
5. `05-page-templates.md` — список шаблонов и состав блоков.
6. `06-api-sync-and-cache.md` — загрузка, кеш, cron, мок-режим.
7. `07-multilingual-and-seo.md` — языки, URL под папками, SEO.
8. `08-mvp-build.md` — первый рабочий MVP.
9. `09-integration-and-expansion.md` — подключение реального API и расширение страниц.
10. `10-testing-deploy-handoff.md` — тесты, деплой, документация.

## Важные решения на согласование

- Мультиязычность: Polylang Pro или WPML. Для папок `/ru/`, `/en/`, `/es/`, `/pt/`, `/fr/`, `/de/` подходят оба варианта.
- Данные API: хранить важные сущности локально в WordPress, а часто меняющиеся данные кешировать отдельно.
- Букмекеры: делать и список, и отдельные страницы под каждого букмекера.
- Игроки/команды/турниры: делать отдельные SEO-страницы, не только архивы.
- Новости: часть ручная, часть потенциально связанная с командами/игроками/турнирами.
